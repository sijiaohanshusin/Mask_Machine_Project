# MaskMachine_H743 Firmware

Initial STM32H743IITx firmware skeleton for the mask dispensing machine.

## Phase 1 Goals

- Boot on STM32H743IIT6 with FreeRTOS.
- Provide UART boot logs.
- Blink PB0/PB1 heartbeat LEDs.
- Create application tasks and queues for dispenser, UI, diagnostics, and polling loops.
- Define replaceable BSP, driver, and service interfaces.
- Bring up the NECCS-proven SDRAM, LTDC RGB565 LCD, DMA2D clocking, and GT9XXX touch path.
- Render the first LVGL mask machine panel and drive a touch-based demo dispense flow.

## Deliberately Deferred

- Real motor/servo/stepper pin assignment.
- RFID module pin assignment and protocol selection.
- Environmental sensor wiring and protocol.
- Wi-Fi/cloud module integration.
- Vision or AI recognition.

## Directory Layout

- `Core/`: generated by STM32CubeMX.
- `Drivers/`: generated HAL/CMSIS code from STM32CubeMX.
- `Middlewares/Third_Party/FreeRTOS/`: generated by STM32CubeMX.
- `Middlewares/LVGL/`: LVGL core copied from the H7 reference project.
- `User/BSP/`: board support wrappers, including SDRAM, LTDC display, and GT9XXX touch.
- `User/Drivers/`: hardware driver abstractions and stubs.
- `User/Services/`: business-level services.
- `User/App/`: task creation, queues, and application bootstrap.

## Regeneration Notes

Open `MaskMachine_H743.ioc` in STM32CubeMX with `File > Load Project` or by double-clicking the `.ioc`. Do not use CubeMX `New Project` for this file; that flow can treat the `.ioc` path as an MCU name and report a missing `.ioc.xml`.

Regenerate code for `MDK-ARM V5` and preserve user code sections. After regeneration, run:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\update_keil_project.ps1
```

The script re-adds the `User/` groups, HAL display driver sources, LVGL source group, include paths, and `LV_CONF_INCLUDE_SIMPLE` define to the Keil project.

## Verification

Command-line Keil build currently completes with `0 Error(s), 30 Warning(s)`. The warnings are from upstream LVGL 8.2.0 source under ARMCC 5; no project-local warnings remain in the phase 1 framework.
